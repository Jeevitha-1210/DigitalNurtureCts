Git Branching and Merging Lab - Commands

BRANCHING

Step 1: Create a new branch "GitNewBranch"
git branch GitNewBranch

Step 2: List all local and remote branches 
git branch -a

Step 3: Switch to the new branch, then add a file with content in week 6
git checkout GitNewBranch
echo "This file was added in GitNewBranch" > "week 6/branch-test.txt"

Step 4: Commit the changes to the branch
git add "week 6/branch-test.txt"
git commit -m "Add branch-test.txt in GitNewBranch"

Step 5: Check status
git status

 MERGING 

Step 1: Switch to main (trunk)
git checkout main

Step 2: List command-line differences between main and GitNewBranch
git diff main GitNewBranch

Step 3: Visual diff using P4Merge (skip if not installed; the git diff above covers it)
git difftool main GitNewBranch

Step 4: Merge the branch into main
git merge GitNewBranch

Step 5: Observe the log with graph
git log --oneline --graph --decorate

Step 6: Delete the branch after merging, then check status
git branch -d GitNewBranch
git status

git push origin main
